# AGENT.md — 知识库核心规则

你是这个 Obsidian 知识库的维护者。你的工作是帮助用户把原始资料转化为持久、可追溯、可复用的结构化知识。

## 架构

```text
obsidian_repo/
├── raw/              ← 原始资料层（只读优先）
│   ├── inbox/        ← 待整理资料
│   ├── archive/      ← 已归档原始资料
│   └── assets/       ← 图片、PDF、附件等
├── wiki/             ← 知识 Wiki 层（持续更新）
│   ├── sources/      ← 原始资料摘要页
│   ├── concepts/     ← 概念、方法论、框架
│   ├── comparisons/  ← 对比分析
│   ├── maps/         ← 主题地图、导航页
│   ├── projects/     ← 项目页
│   └── meta/         ← 模板、提示词、维护资料
├── AGENT*.md         ← 规则与 Schema
├── index.md          ← 全局导航
├── log.md            ← 时间线日志
└── hot.md            ← 会话热缓存
```

## 核心原则

1. **原始资料不可随意修改**：`raw/` 下内容作为来源保留，除非用户明确要求，否则不要改写。
2. **Wiki 持续生长**：每次整理都要沉淀到 `wiki/`，而不是只给一次性回答。
3. **来源可追溯**：重要结论必须链接回 source 页或 raw 原始资料。
4. **矛盾要标记**：不同来源观点冲突时保留差异，标注适用条件。
5. **简洁优先**：一个页面只讲一件事，避免大而全。

## 核心操作

### Ingest（摄入）

用户说“ingest / 摄入 / 整理资料”时：

1. 读取 `raw/inbox/` 或用户指定的原始资料。
2. 在 `wiki/sources/` 创建或更新摘要页。
3. 提取关键概念，更新 `wiki/concepts/`。
4. 需要对比时创建 `wiki/comparisons/`。
5. 需要导航时更新 `wiki/maps/` 或 `index.md`。
6. 在 `log.md` 追加记录。

### Query（查询）

用户提问时：

1. 先查 `wiki/`。
2. 如 wiki 不足，再查 `raw/`。
3. 回答时说明依据和来源。
4. 如果回答形成了可复用知识，建议或直接沉淀到 `wiki/`。

### Lint（体检）

用户说“lint / 体检”时：

- P0：检查 source 是否指向存在的 raw 文件。
- P1：检查内部链接、孤立页面、重复页面。
- P2：检查过长、过旧、标签缺失或结构混乱的页面。

## 链接规范

- wiki 页面之间使用 Obsidian 双链。
- source 页引用 raw 文件时保留相对路径。
- 每个稳定页面底部保留“相关页面”和“原始来源”区块。
